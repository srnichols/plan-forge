## How they got hidden

`.github/workflows/validate.yml` ran `setup.ps1` / `setup.sh` against test projects but never `cd pforge-mcp; npx vitest run`. Test results were only visible to maintainers running locally. v2.82.2 added a `pforge-mcp-tests` CI job with `BASELINE_FAILURES=35` tolerance so:

- New regressions (failure count > 35) hard-fail the build
- The 31 documented failures don't block PRs
- Drop the tolerance number as each bucket is fixed; 0 means fully green
